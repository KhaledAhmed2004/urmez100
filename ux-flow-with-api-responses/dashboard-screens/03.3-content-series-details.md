# Screen 3.3: Content Management - Series Details (Episode Management)

> **Section**: Dashboard APIs (Admin-Facing)
> **Base URL**: `{{baseUrl}}` = `http://localhost:5000/api/v1`
> **Response format**: See [Standard Response Envelope](../README.md#standard-response-envelope)
> **Related screens**: [Content: Series](./03.2-content-series.md)

## UX Flow

### Episode Management Flow

1. Admin clicks "View" on a series in the content management module.
2. Series overview (Title, Poster, Description) and seasons count are fetched → `GET /admin/series/:seriesId/details` (→ 1.1)
3. Episodes for the default or selected season are fetched → `GET /admin/series/:seriesId/episodes?season=1` (→ 2.1)
4. Admin can search for specific episodes by title → `GET /admin/series/:seriesId/episodes?search=The Beginning` (→ 2.1)
5. Episode table renders:
   - **Episode Title**
   - **Duration**
   - **Release Date**
   - **Status** (Published/Draft)
   - **Availability** (Free/Premium plans)
   - **Actions** (Edit, Delete)
6. Changing the season filter refreshes the episode list.
7. Admin adds a new episode:
   - **Step 1 (Media)**: Uploads the episode video and poster.
   - **Step 2 (Metadata)**: Provides title, description, release date, and availability → `POST /admin/series/:seriesId/episodes` (→ 3.1)

---

## Edge Cases & Solutions

| Scenario | Behavior / Solution |
| :--- | :--- |
| **No Episodes** | Table shows "No episodes found for this season". Stats will reflect 0 episodes. |
| **Invalid Series** | Returns 404 Not Found if the series ID does not exist. |
| **Draft Status** | Episodes marked as "Draft" are only visible to admins. They are excluded from public streaming until published. |
| **Auto-Sync Counts** | Adding or deleting an episode automatically updates the `totalEpisodes` and `seasonsCount` on the parent Series document. |

### 1.1 Series Details Overview
```http
GET /admin/series/:seriesId/details
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

> Returns high-level details for a specific series, including the list of seasons and episode counts.

**Business Logic:**
- **Source**: Fetches the parent `Content` document.
- **Seasons**: Returns all seasons linked to this series.
- **Counts**: Includes `totalEpisodes` for the series and `episodeCount` for each individual season.

### 2.1 Get/Search Episodes
```http
GET /admin/series/:seriesId/episodes
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

> Returns a paginated list of episodes for a series, filtered by season ID or search term.

**Query Parameters:**
- `seasonId`: Filter by a specific Season ID (e.g. `664a1b2c3d4e5f6a7b8c9d55`)
- `seasonNumber`: Alternatively, filter by season number (e.g. `1`)
- `search`: Episode title search
- `page`: Pagination page number
- `limit`: Pagination limit

**Response:**
```json
{
  "success": true,
  "statusCode": 200,
  "message": "Episodes list fetched",
  "pagination": { "page": 1, "limit": 10, "total": 8, "totalPage": 1 },
  "data": [
    {
      "_id": "e1",

      "seasonId": "664a1b2c3d4e5f6a7b8c9d55",
      "seasonNumber": 1,
```

### 3.1 Create Episode
```http
POST /admin/series/:seriesId/episodes
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
Content-Type: multipart/form-data
```

> Adds a new episode to a specific series.

**Business Logic:**
- **Uploads**: Handles multipart upload for `videoFile` and `thumbnailFile`.
- **Validation**: Ensures the parent series exists before creation.
- **Sync**: After creation, it recalculates the series' `totalEpisodes` and `seasonsCount` (based on the highest season number found).

**Request Body (Multipart):**
- `title`: String
- `description`: String
- `seasonNumber`: Number
- `duration`: Number (minutes)
- `releaseDate`: Date String
- `availability`: `FREE` | `WEEKLY` | `MONTHLY` | `YEARLY` | `ALL`
- `isDraft`: Boolean
- `videoFile`: File
- `thumbnailFile`: File
