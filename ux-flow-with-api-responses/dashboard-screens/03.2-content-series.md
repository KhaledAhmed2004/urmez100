## UX Flow

### Series Management Flow

1. Admin clicks on the "Content Management > Series" module from the sidebar.
2. Series stats cards are fetched on page load → `GET /admin/series/stats` (→ 3.2.1)
3. Series list is fetched on page load → `GET /admin/series` (→ 3.2.2)
4. Admin uses the search bar to search for a series by title → `GET /admin/series?search=The Witcher` (→ 3.2.2)
5. Series table renders:
   - **Series** (Title & Poster)
   - **Seasons** (Count)
   - **Status** (Published/Draft)
   - **Subscribe** (Weekly/Monthly/Yearly/Free)
   - **Actions** (Edit, Delete, View Episodes)
6. Admin clicks "View" to go to the series episodes management page → [Content: Series Details](./03.3-content-series-details.md)
7. Admin clicks "Add Series" to create a new series entry → `POST /admin/series` (→ 3.2.3)
8. Admin edits a series metadata or images → `PATCH /admin/series/:seriesId` (→ 3.2.4)
9. Admin deletes a series (and all its episodes) → `DELETE /admin/series/:seriesId` (→ 3.2.5)
10. Admin toggles series status (Draft/Published) → `PATCH /admin/series/:seriesId/status` (→ 3.2.6)

## Edge Cases & Solutions

| Scenario | Behavior / Solution |
| :--- | :--- |
| **No Series** | Stats cards will return zero, and the table will show "No data found". |
| **Search No Result** | Returns 200 OK but with an empty data array and 0 total count in pagination. |
| **Draft Series** | Frontend shows a "Draft" badge. Draft series are excluded from the public app's content listings. |
| **Season Management** | The `seasonsCount` is managed at the series level. Adding/removing episodes in the details screen updates this count. |
| **Cascade Delete** | Deleting a series automatically deletes all its associated episodes from the database to prevent orphaned records. |

### 3.2.1 Series Stats (Overview Cards)
```http
GET /admin/series/stats
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

> For the summary stat cards in the dashboard's series section.

**Business Logic:**
- **Total Series**: Counts all `Content` documents where `type: 'SERIES'`. Growth is calculated by comparing the current month's total against the last month.
- **Total Likes**: Calculated using `views` as a proxy from all series content. Growth is calculated against the previous month.
- **CTR**: Click-Through Rate proxy based on total views. Growth follows the views growth.
- **Total Views**: Sum of the `views` field from all `Content` documents where `type: 'SERIES'`. Growth is calculated against the previous month.
- **Realistic Fallback**: If the database is empty, the API returns realistic "demo" values (e.g., 1,450 series, 120.5M views) to facilitate frontend development.

### 3.2.2 Get/Search Series List
```http
GET /admin/series
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

> Handles series list search, filtering, and pagination.

**Business Logic:**
- **Source**: Fetches `Content` documents where `type: 'SERIES'`.
- **Search**: Supports partial matching on the `title` field.
- **Filtering**: Allows filtering by `planStatus` (Subscription type) and `status` (PUBLISHED/DRAFT).
- **Mapping**: Returns a flattened data structure including `seasonsCount` and `subscriptionType` (from `planStatus`).

### 3.2.3 Create Series
```http
POST /admin/series
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
Content-Type: application/json
```

> Creates a new series entry. Media files and episodes are added later.

**Business Logic:**
- **Fields**: Accepts `title`, `description`, `releaseYear`, `genres`, `cast`, and `isDraft`.
- **Draft Mode**: Can be created as a `DRAFT` (hidden from users) or `PUBLISHED`.
- **Initialization**: Seasons and episodes are initialized at `0`.

**Request Body:**
```json
{
  "title": "The Witcher",
  "description": "Geralt of Rivia, a solitary monster hunter...",
  "releaseYear": 2019,
  "genres": ["664a1b2c3d4e5f6a7b8c9d01", "664a1b2c3d4e5f6a7b8c9d02"],
  "cast": ["Henry Cavill", "Anya Chalotra"],
  "isDraft": true
}
```

### 3.2.4 Update Series
```http
PATCH /admin/series/:seriesId
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
Content-Type: multipart/form-data
```

> Updates series metadata, cast, or media files (Poster/Trailer).

**Business Logic:**
- **Partial Updates**: Only fields provided in the request body are updated.
- **Media**: Handles file uploads for `trailerFile` and `posterFile`.
- **Normalization**: Ensures `genres` and `cast` are stored as arrays.

---

### 3.2.5 Delete Series
```http
DELETE /admin/series/:seriesId
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

> Permanently removes a series and all its episodes.

**Business Logic:**
- **Cascade Deletion**: This is a destructive operation that first deletes all episodes associated with the `seriesId` in the `Episode` collection, then deletes the series record itself.

---

### 3.2.6 Update Series Status
```http
PATCH /admin/series/:seriesId/status
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
Content-Type: application/json
```

> Toggles the visibility of the series (Draft vs Published).

**Request Body:**
```json
{
  "status": "PUBLISHED"
}
```
