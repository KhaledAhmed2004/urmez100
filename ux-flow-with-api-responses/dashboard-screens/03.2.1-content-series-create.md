# Screen 3.2.1: Create Series & Seasons (Step-by-Step)

This guide explains how to create a series, define its seasons, and add episodes under those seasons.

---

## 1. Overview of the Hierarchy

The series content follows a 3-level hierarchy:
1.  **Series**: The main show (e.g., "The Witcher"). Contains general metadata (Title, Description, Genres, Cast, etc.).
2.  **Season**: A grouping within the series (e.g., "Season 1"). Has its own Title and Poster.
3.  **Episode**: Individual video files under a specific Season.

---

## 2. Phase 1: Create Series Metadata

First, create the main series container. This endpoint now only handles basic metadata.

### **Endpoint**
```http
POST /admin/series
Content-Type: application/json
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

### **Request Body (JSON)**

| Field | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `title` | `string` | Yes | Series title. |
| `description` | `string` | Yes | Series description. |
| `genres` | `string[]` | Yes | Array of Genre IDs. |
| `cast` | `string[]` | No | Array of cast names. |
| `releaseYear` | `number` | Yes | Release year. |
| `isDraft` | `boolean` | Yes | `true` for Draft, `false` to Publish immediately. |

---

## 3. Phase 2: Upload Series Media (Optional)

After creation, you can upload the main series poster and trailer using the update endpoint.

### **Endpoint**
```http
PATCH /admin/series/:seriesId
Content-Type: multipart/form-data
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

| Field | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `posterFile` | `file` | No | Main series poster (Vertical). |
| `trailerFile` | `file` | No | Series teaser/trailer video. |

---

## 4. Phase 3: Define Seasons

Once the series is created, you can add seasons to it. Each season acts as a folder for episodes and can have its own trailer/teaser.

### **Endpoint**
```http
POST /admin/series/:seriesId/seasons
Content-Type: multipart/form-data
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

### **Request Fields (Form-Data)**

| Field | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `title` | `string` | Yes | Season name (e.g., "Season 1: The Beginning"). |
| `seasonNumber` | `number` | Yes | The numeric order (e.g., 1, 2). |
| `posterFile` | `file` | Yes | Poster specific to this season. |
| `trailerFile` | `file` | No | Teaser or trailer video for this specific season. |

---

## 5. Phase 4: Add Episodes to a Season

Episodes are uploaded using the same Multipart strategy as movies, but they are linked to a **Season ID**.

### **Step 1: Upload Video to R2**
Use the standard multipart upload flow to get a Bunny CDN URL for the episode video.
(Refer to [Movie Upload Strategy](../video-upload-guide.md))

### **Step 2: Save Episode Metadata**
```http
POST /admin/series/seasons/:seasonId/episodes
Content-Type: application/json
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

### **Request Body**
```json
{
  "title": "The First Journey",
  "description": "The protagonist starts their adventure...",
  "episodeNumber": 1,
  "duration": 45,
  "videoUrl": "https://uremz-video-stream.b-cdn.net/...",
  "isDraft": false
}
```

---

## 6. Summary Flow for Developers

1.  **POST `/admin/series`** → Save Series Metadata → Get `seriesId`.
2.  **PATCH `/admin/series/:seriesId`** → (Optional) Upload Series Poster/Trailer.
3.  **POST `/admin/series/:seriesId/seasons`** → Create Season(s) → Get `seasonId`.
4.  **POST `/admin/series/seasons/:seasonId/episodes`** → Add Episodes to that season.
