## UX Flow

### Movies Management Flow

1. Admin clicks on the "Content Management > Movies" module from the sidebar.
2. Movies stats cards are fetched on page load → `GET /admin/movies/stats` (→ 3.1)
3. Movies list is fetched on page load → `GET /admin/movies` (→ 3.2)
4. Admin uses the search bar to search for a movie by title → `GET /admin/movies?search=Inception` (→ 3.2)
5. Admin uses filters to select plan status or general status → `GET /admin/movies?planStatus=MONTHLY&status=PUBLISHED` (→ 3.2)
6. Movies table renders:
   - **Title**
   - **Image** (Poster)
   - **Duration**
   - **Status** (PUBLISHED/DRAFT)
7. Admin clicks "View Details" from the movie table row → [Content: Movie Details](./03.1-content-movies-details.md)
8. Admin clicks "Upload New Movie" and fills the form to add a movie:
   - **Step 1 (Video Upload)**: Uses S3 Multipart upload to securely upload the video path → `POST /admin/movies/upload/initiate` & `complete` (→ 3.7)
   - **Step 2 (Metadata)**: Creates the movie using the uploaded video URL and other metadata → [Create Movie](./03.1-content-movies-create.md)
9. Admin toggles movie status (Publish/Draft) → `PATCH /admin/movies/:movieId/status` (→ 3.6)
10. Confirm modal appears on clicking delete, then submit → `DELETE /admin/movies/:movieId` (→ 3.5)
11. Edit action updates movie details → `PATCH /admin/movies/:movieId` (→ 3.4)

---

## Video Upload Strategy (S3 Multipart)

For large video files (Video/Trailer), we use the **AWS S3 Multipart Upload + Presigned URLs** strategy. Detailed guide here: [Video Upload Guide](../video-upload-guide.md).

### Summary Flow:
1. `POST /admin/movies/upload/initiate` → returns `uploadId` and `key`.
2. `POST /admin/movies/upload/presigned-urls` → generates signed URLs for chunks.
3. Chunks are uploaded directly to S3 from the frontend.
4. `POST /admin/movies/upload/complete` → joins all chunks and returns the final `location`.

## Edge Cases & Solutions

| Scenario | Behavior / Solution |
| :--- | :--- |
| **No Movies** | Stats cards will return zero, and the table will show "No data found". |
| **Invalid Movie ID** | Returns 404 Not Found for PATCH/DELETE requests with an invalid ID. |
| **Search No Result** | Returns 200 OK but with an empty data array and 0 total count in pagination. |
| **CTR Calculation** | `ctr` returns 0 from the server if `totalViews` is zero. |
| **Upload Failure** | Incomplete parts will be automatically deleted via bucket lifecycle policies if multipart upload fails. |

### 3.1 Movies Stats (Overview Cards)
```http
GET /admin/movies/stats
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

> For the summary stat cards in the dashboard's movies section. Returns total movies, likes, views, and Click-through Rate (CTR).

**Business Logic:**
- **Total Movies**: Counts all `Content` documents where `type: 'MOVIE'`. Uses `AggregationBuilder` to calculate monthly growth.
- **Total Likes**: Currently uses the `views` field as a proxy for engagement/likes. Growth is calculated against the previous month.
- **CTR**: Click-Through Rate. Currently hardcoded to `25%` if the movie has views, otherwise `0`. It includes growth tracking based on views.
- **Total Views**: Sum of the `views` field from all `Content` documents where `type: 'MOVIE'`.
- **Fallback Logic**: If no real data exists, the API returns demo values (e.g., 12,450 movies, 89.5M views) to ensure the frontend displays a populated UI during initial development.

**Implementation:**
- **Route**: [admin.route.ts](file:///src/app/modules/admin/admin.route.ts)
- **Controller**: [admin.controller.ts](file:///src/app/modules/admin/admin.controller.ts) — `getMoviesStats`
- **Service**: [admin.service.ts](file:///src/app/modules/admin/admin.service.ts) — `getMoviesStats`

**Field Reference:**

| Field | Type | Description |
| :--- | :--- | :--- |
| `meta.comparisonPeriod` | `string` | Always `"month"` — current vs last calendar month |
| `{metric}.value` | `number \| string` | Total count as of now (formatted with K/M suffixes) |
| `{metric}.changePct` | `number` | Always a positive magnitude (e.g. `25`, `7.14`). Use `direction` for sign. |
| `{metric}.direction` | `"up" \| "down" \| "neutral"` | `"up"` = growth, `"down"` = decline, `"neutral"` = no change |

#### Response
```json
{
  "success": true,
  "statusCode": 200,
  "message": "Movies stats retrieved successfully",
  "data": {
    "meta": {
      "comparisonPeriod": "month"
    },
    "totalMovies": {
      "value": 12450,
      "changePct": 0.5,
      "direction": "up"
    },
    "totalLikes": {
      "value": "8.9M",
      "changePct": 0.5,
      "direction": "up"
    },
    "ctr": {
      "value": 25,
      "changePct": 5.2,
      "direction": "up"
    },
    "totalViews": {
      "value": "89.5M",
      "changePct": 0.5,
      "direction": "up"
    }
  }
}
```

### 3.2 Get/Search Movies List
```http
GET /admin/movies
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

> Handles movies list search, filtering, and pagination. Table columns mapping: Title, Image, Duration, Status.

**Business Logic:**
- **Source**: Fetches `Content` documents where `type: 'MOVIE'`.
- **Search**: Supports case-insensitive partial matching on the `title` field using the `QueryBuilder`.
- **Filtering**: Supports filtering by `planStatus` and `status`.
- **Mapping**: Transforms the raw data into a frontend-friendly format:
    - `duration`: Formatted from minutes to `Xh Ym` (e.g., `2h 28m`).
    - `planStatus`: Included for the plan badges in the table.

**Implementation:**
- **Route**: [admin.route.ts](file:///src/app/modules/admin/admin.route.ts)
- **Controller**: [admin.controller.ts](file:///src/app/modules/admin/admin.controller.ts) — `getAdminMovies`
- **Service**: [admin.service.ts](file:///src/app/modules/admin/admin.service.ts) — `getAdminMoviesList`

**Query Parameters:**

| Parameter | Description | Default |
| :--- | :--- | :--- |
| `search` | Partial match on Title | — |
| `planStatus` | Filter by Plan (`FREE`, `WEEKLY`, `MONTHLY`, `YEARLY`, `ALL`) | — |
| `status` | Filter by Status (`PUBLISHED`, `DRAFT`, `ALL`) | — |
| `page` | Pagination page number | `1` |
| `limit` | Pagination limit | `10` |

#### Response
```json
{
  "success": true,
  "statusCode": 200,
  "message": "Movies list fetched",
  "pagination": { "page": 1, "limit": 10, "total": 450, "totalPage": 45 },

      "poster": "https://example.com/images/interstellar.jpg",
      "duration": "2h 49m",
      "status": "PUBLISHED",
      "planStatus": ["MONTHLY", "YEARLY"]
```

### 3.3 Create Movie
```http
POST /admin/movies
Content-Type: application/json
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

> Creates a movie using metadata and URLs (video, trailer, etc.) obtained from multipart uploads.

**Business Logic:**
- **Type**: Automatically sets `type: 'MOVIE'`.
- **Data Normalization**: 
    - `genres`: Ensures genres are stored as an array.
    - `duration`/`releaseYear`/`rating`/`views`: Converts string inputs from multipart forms to numbers.
    - `planStatus`: Maps the `availability` field from the frontend to `planStatus`.
    - `status`: Maps `isDraft` boolean to `DRAFT` or `PUBLISHED` status.

**Implementation:**
- **Route**: [admin.route.ts](file:///src/app/modules/admin/admin.route.ts)
- **Controller**: [admin.controller.ts](file:///src/app/modules/admin/admin.controller.ts) — `createMovie`
- **Service**: [admin.service.ts](file:///src/app/modules/admin/admin.service.ts) — `createMovieToDB`

**Request Body:**
```json
{
  "title": "Inception",
  "description": "A thief who steals corporate secrets...",
  "genres": ["Sci-Fi", "Action"],
  "duration": 148,
  "releaseYear": 2010,
  "rating": 8.8,
  "videoUrl": "https://bucket.s3.region.amazonaws.com/videos/inception.mp4",
  "trailerUrl": "https://bucket.s3.region.amazonaws.com/trailers/inception-trailer.mp4",
  "poster": "https://bucket.s3.region.amazonaws.com/posters/inception-poster.jpg",
  "thumbnail": "https://bucket.s3.region.amazonaws.com/thumbnails/inception-thumb.jpg",
  "availability": "MONTHLY",
  "isDraft": false
}
```

#### Response
```json
{
  "success": true,
  "statusCode": 201,
  "message": "Movie created successfully",
  "data": {
    "_id": "664a1b2c3d4e5f6a7b8c9d1a",
    "title": "Inception",
    "status": "PUBLISHED"
  }
}
```

### 3.5 Delete Movie
```http
DELETE /admin/movies/:movieId
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

> Permanently deletes the movie.

**Business Logic:**
- **Destruction**: This is a hard delete operation using `findByIdAndDelete`.
- **Cleanup**: Media files in S3 are not deleted by this API; cleanup should be handled via lifecycle policies or separate processes.

### 3.7 S3 Multipart Upload Endpoints

These endpoints are used for uploading large video and trailer files.

#### A. Initiate Upload
`POST /admin/movies/upload/initiate`
- **Request**: `{ "fileName": "vid.mp4", "contentType": "video/mp4" }`
- **Response**: `{ "success": true, "data": { "uploadId": "...", "key": "..." } }`

#### B. Get Presigned URLs
`POST /admin/movies/upload/presigned-urls`
- **Request**: `{ "uploadId": "...", "key": "...", "partNumbers": [1, 2, 3] }`
- **Response**: `{ "success": true, "data": [{ "partNumber": 1, "url": "..." }, ...] }`

#### C. Complete Upload
`POST /admin/movies/upload/complete`
- **Request**: `{ "uploadId": "...", "key": "...", "parts": [{ "ETag": "...", "PartNumber": 1 }] }`
- **Response**: `{ "success": true, "data": { "location": "https://...", "key": "..." } }`
