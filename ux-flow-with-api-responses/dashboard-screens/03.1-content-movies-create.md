# Screen 3.1: Create Movie (Step-by-Step Integration)

This guide explains the detailed process and API requirements for creating a new movie in the dashboard.

---

## 1. Overview of the Creation Flow

The movie creation process is divided into two main phases:
1.  **Media Upload (Multipart)**: Uploading large files (Video & Trailer) to Cloudflare R2 via the backend.
2.  **Metadata Submission**: Sending the final movie details (Title, Description, URLs, etc.) to the database.

---

## 2. Phase 1: Media Upload (S3 Multipart)

For large video files (Movie Video & Teaser), we use a multipart upload strategy. This is required for files larger than 5MB to ensure stability and performance.

### **Step 1: Initiate Upload**
Get an `uploadId` and a unique `key` from the backend.
- **Endpoint**: `POST /admin/movies/upload/initiate`
- **Payload**: `{ "fileName": "movie.mp4", "contentType": "video/mp4" }`
- **Response**: `{ "uploadId": "...", "key": "..." }`

### **Step 2: Slice File & Get Presigned URLs**
Divide the file into chunks (Minimum 5MB per chunk). For each chunk, request a presigned URL.
- **Endpoint**: `POST /admin/movies/upload/presigned-urls`
- **Payload**: `{ "uploadId": "...", "key": "...", "partNumbers": [1, 2, 3] }`
- **Response**: `[{ "partNumber": 1, "url": "..." }, ...]`

### **Step 3: Upload Chunks Directly to R2**
Send a `PUT` request to each presigned URL with the binary chunk.
- **Method**: `PUT`
- **Headers**: `Content-Type: video/mp4`
- **Critical**: Save the `ETag` from the response headers for each part.

### **Step 4: Complete Upload**
Notify the backend to merge the chunks and get the final URL.
- **Endpoint**: `POST /admin/movies/upload/complete`
- **Payload**: `{ "uploadId": "...", "key": "...", "parts": [{ "ETag": "...", "PartNumber": 1 }, ...] }`
- **Result**: Returns the final **Bunny CDN URL** (e.g., `https://uremz-video-stream.b-cdn.net/...`).

## 3. Implementation Code Snippet (Axios)

Share this logic with the developer for a clean integration:

```javascript
const CHUNK_SIZE = 5 * 1024 * 1024; // 5MB

async function uploadMedia(file) {
  // 1. Initiate
  const { data: { data: { uploadId, key } } } = await axios.post('/admin/movies/upload/initiate', {
    fileName: file.name, contentType: file.type
  });

  const totalParts = Math.ceil(file.size / CHUNK_SIZE);
  const parts = [];

  for (let i = 1; i <= totalParts; i++) {
    const chunk = file.slice((i - 1) * CHUNK_SIZE, i * CHUNK_SIZE);
    
    // 2. Get URL
    const { data: { data: urls } } = await axios.post('/admin/movies/upload/presigned-urls', {
      uploadId, key, partNumbers: [i]
    });

    // 3. PUT Chunk
    const res = await axios.put(urls[0].url, chunk, { headers: { 'Content-Type': file.type } });
    parts.push({ ETag: res.headers.etag.replace(/"/g, ''), PartNumber: i });
  }

  // 4. Complete
  const { data: { data: { location } } } = await axios.post('/admin/movies/upload/complete', {
    uploadId, key, parts
  });

  return location; // Use this in Phase 2
}
```

## 4. Phase 2: Metadata Submission (Create Movie)

Once you have the URLs for the video and trailer, call the following endpoint:

### **Endpoint**
```http
POST /admin/movies
Content-Type: application/json
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

### **Request Body Fields**

| Field | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `title` | `string` | Yes | The title of the movie. |
| `description` | `string` | Yes | Detailed plot or description of the movie. |
| `genres` | `string[]` | Yes | Array of Genre IDs (ObjectIds). |
| `cast` | `string[]` | No | Array of cast member names. |
| `releaseYear` | `number` | Yes | Year of release (e.g., 2024). |
| `duration` | `number` | Yes | Total duration in **minutes**. |
| `availability` | `string[]` | Yes | Array of plans: `FREE`, `WEEKLY`, `MONTHLY`, `YEARLY`, or `ALL`. |
| `videoUrl` | `string` | Yes | The final Bunny CDN URL obtained from Phase 1. |
| `trailerUrl` | `string` | No | The final Bunny CDN URL for the movie teaser/trailer. |
| `poster` | `string` | No | URL of the vertical movie poster. |
| `isDraft` | `boolean` | Yes | `true` for Draft, `false` to Publish immediately. |
| `isPremium` | `boolean` | No | Mark if the movie is a premium/VIP content. |
| `isRecent` | `boolean` | No | Mark if it should appear in the "Recent" section. |

### **Example Payload**
```json
{
  "title": "The Grand Heist",
  "description": "A group of skilled thieves plan the ultimate robbery.",
  "genres": ["664a1b2c3d4e5f6a7b8c9d11", "664a1b2c3d4e5f6a7b8c9d12"],
  "cast": ["John Doe", "Jane Smith", "Robert Brown"],
  "releaseYear": 2024,
  "duration": 125,
  "availability": ["MONTHLY", "YEARLY"],
  "videoUrl": "https://uremz-video-stream.b-cdn.net/media/videos/1715856000-movie.mp4",
  "trailerUrl": "https://uremz-video-stream.b-cdn.net/media/videos/1715856000-teaser.mp4",
  "poster": "https://pub-xxxx.r2.dev/posters/movie-poster.jpg",
  "isDraft": false,
  "isPremium": true,
  "isRecent": true
}
```

## 5. Business Logic & Normalization (Backend)

When the `POST /admin/movies` request is received, the backend performs the following:
1.  **Normalization**: 
    - Converts `duration`, `releaseYear`, and `rating` to numbers.
    - Ensures `genres` and `cast` are always arrays (even if sent as strings).
    - Maps `isDraft: true` to `status: 'DRAFT'` and `isDraft: false` to `status: 'PUBLISHED'`.
    - Maps `availability` to the internal `planStatus` field.
2.  **Type Assignment**: Automatically sets the content `type` to `'MOVIE'`.
3.  **Storage**: Saves the document to the `Content` collection.
