# Screen 3.3.1: Create Series Episode (Step-by-Step Integration)

This guide explains the detailed process and API requirements for adding a new episode to a series under a specific season.

---

## 1. Overview of the Creation Flow

The episode creation process follows the same two-phase logic as movies:
1.  **Media Upload (Multipart)**: Uploading the episode video to Cloudflare R2.
2.  **Metadata Submission**: Linking the video URL to a specific **Season** and **Series**.

---

## 2. Phase 1: Media Upload (S3 Multipart)

For episode videos, we use the same multipart upload strategy.

### **Step 1: Initiate Upload**
Get an `uploadId` and a unique `key` from the backend.
- **Endpoint**: `POST /admin/movies/upload/initiate`
- **Payload**: `{ "fileName": "episode1.mp4", "contentType": "video/mp4" }`
- **Response**: `{ "uploadId": "...", "key": "..." }`

### **Step 2: Slice File & Get Presigned URLs**
Divide the file into chunks (Minimum 5MB per chunk).
- **Endpoint**: `POST /admin/movies/upload/presigned-urls`
- **Payload**: `{ "uploadId": "...", "key": "...", "partNumbers": [1, 2, 3] }`

### **Step 3: Upload Chunks Directly to R2**
Send a `PUT` request to each presigned URL with the binary chunk and save the `ETag`.

### **Step 4: Complete Upload**
Notify the backend to merge the chunks and get the final **Bunny CDN URL**.
- **Endpoint**: `POST /admin/movies/upload/complete`
- **Result**: Returns the final URL (e.g., `https://uremz-video-stream.b-cdn.net/media/videos/...`).

---

## 3. Implementation Code Snippet (Axios)

```javascript
const CHUNK_SIZE = 5 * 1024 * 1024; // 5MB

async function uploadEpisodeVideo(file) {
  // 1. Initiate
  const { data: { data: { uploadId, key } } } = await axios.post('/admin/movies/upload/initiate', {
    fileName: file.name, contentType: file.type
  });

  const totalParts = Math.ceil(file.size / CHUNK_SIZE);
  const parts = [];

  for (let i = 1; i <= totalParts; i++) {
    const chunk = file.slice((i - 1) * CHUNK_SIZE, i * CHUNK_SIZE);
    
    // 2. Get URL
    const { data: { data: urls } } = await axios.post('/admin/movies/upload/presigned-urls', {
      uploadId, key, partNumbers: [i]
    });

    // 3. PUT Chunk
    const res = await axios.put(urls[0].url, chunk, { headers: { 'Content-Type': file.type } });
    parts.push({ ETag: res.headers.etag.replace(/"/g, ''), PartNumber: i });
  }

  // 4. Complete
  const { data: { data: { location } } } = await axios.post('/admin/movies/upload/complete', {
    uploadId, key, parts
  });

  return location; // Final Bunny CDN URL
}
```

## 4. Phase 2: Metadata Submission (Create Episode)

Once you have the video URL, submit the episode metadata. Note that the episode must be linked to a **Series ID** (in the URL) and a **Season ID** (in the body).

### **Endpoint**
```http
POST /admin/series/:seriesId/episodes
Content-Type: application/json
Authorization: Bearer {{accessToken}} (SUPER_ADMIN)
```

### **Request Body Fields**

| Field | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `title` | `string` | Yes | Episode title. |
| `description` | `string` | Yes | Brief description of the episode. |
| `seasonId` | `string` | Yes | The ID of the Season this episode belongs to (Required for filtering). |
| `episodeNumber` | `number` | Yes | The order of the episode (e.g., 1, 2, 3) (Required for sorting). |
| `seasonNumber` | `number` | No | Optional. If not provided, backend will auto-set this based on the `seasonId`. |
| `duration` | `number` | Yes | Duration in **minutes**. |
| `videoUrl` | `string` | Yes | The final Bunny CDN URL obtained from Phase 1. |
| `thumbnail` | `string` | No | URL of the episode thumbnail (usually uploaded via backend). |
| `availability` | `string` | No | `FREE`, `WEEKLY`, `MONTHLY`, `YEARLY`, `ALL`. |
| `isDraft` | `boolean` | Yes | `true` for Draft, `false` to Publish. |

### **Example Payload**
```json
{
  "title": "The Journey Begins",
  "description": "Geralt enters the town of Blaviken.",
  "seasonId": "664a1b2c3d4e5f6a7b8c9d55",
  "episodeNumber": 1,
  "duration": 61,
  "videoUrl": "https://uremz-video-stream.b-cdn.net/media/videos/1715856000-ep1.mp4",
  "thumbnail": "https://pub-xxxx.r2.dev/thumbnails/ep1-thumb.jpg",
  "availability": "FREE",
  "isDraft": false
}
```

---

## 5. Business Logic & Normalization (Backend)

1.  **Validation**: Verifies that the `seriesId` and `seasonId` exist.
2.  **Normalization**: 
    - Converts `duration` and `episodeNumber` to numbers.
    - Sets `seasonNumber` automatically based on the selected Season's record.
    - Maps `isDraft` to `status: 'DRAFT'` or `'PUBLISHED'`.
3.  **Sync**: Automatically increments `totalEpisodes` in the parent Series document.
